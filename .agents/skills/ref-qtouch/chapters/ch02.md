# ch02 — 技术特性

## 2.1 核心架构

### iFIX 架构

```
WorkSpace (IDE) → SAC (Scan, Alarm, Control) → 通信层 (KEPServerEX/IGS) → Historian
```

关键组件：
- **SAC 程序**: 数据库链执行、I/O 扫描、报警检测、控制输出
- **数据库链**: 模拟量输入/输出链、数字量输入/输出链，包含报警检测、量程转换、滤波、PID
- **报警管理**: 分区、路由、升级、搁置、ISA 18.2 合规

### CIMPLICITY 架构

基于节点的分布式架构：Server + Client + WebServer + Active Update

### QTouch 架构

四维一体：云、网、端、边
- 基于 IEC 61131-3 的 SoftPLC（梯形图编程）
- C 语言编辑编译系统
- Qt 跨平台技术栈
- 嵌入式设备部署能力

## 2.2 关键技术能力

### 高性能 HMI（ISA 101）
- 异常驱动显示（Abnormal Situation Display）
- 减少视觉噪音（灰度/低饱和度设计）
- 层级导航（Drill-down）
- 500+ Dynamo 预置对象

### 报警管理
- ISA 18.2 标准合规
- 优先级/严重性分级
- 分区和路由
- 升级、搁置、抑制
- 队列管理（默认 10,000）

### 冗余与高可用
- 故障转移（主/备服务器）
- 数据库同步压缩（65,000 标签 90s→3s）
- 热备切换
- 网络隔离

### 开放集成
- OPC UA/DA, ActiveX/COM, VBA, REST API, SQL, ODBC, MQTT

## 2.3 平台支持

| 产品 | 操作系统 | 部署方式 |
|------|---------|---------|
| iFIX | Windows 10/11 64-bit | 本地、虚拟化、云 |
| CIMPLICITY | Windows Server | 本地、虚拟化 |

## 2.4 通信协议

通过 KEPServerEX/IGS 支持 200+ 设备驱动：Allen-Bradley, Siemens S7, Modbus TCP/RTU, OPC UA/DA, BACnet, MQTT, DNP3
