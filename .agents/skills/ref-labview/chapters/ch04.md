# Chapter 4: RT, FPGA & Advanced Modules

## 4.1 LabVIEW Real-Time Module

LabVIEW RT runs on Phar Lap ETS or NI Linux Real-Time (PREEMPT_RT) on PXI controllers and CompactRIO.

### Dual-Mode Execution

- **Host VI**: Runs on Windows development PC, handles UI
- **RT Target VI**: Runs on RT controller, handles deterministic control logic

Communication: host <-> RT target via TCP/IP (NI-PSP protocol), Shared Variables, or Network Streams.

### Timed Loop

Core RT execution primitive:

- Microsecond-level cycle precision (minimum 1us, hardware-dependent)
- 6 processor allocation modes (auto, dedicated core, multi-core round-robin, etc.)
- Priority levels (1-65535), high-priority loops preempt low-priority
- Watchdog timer: detects loop overrun, prevents system deadlock
- Multi-rate architecture: different Timed Loops can run at different periods

**Jitter characteristics**:
- PXI controller, simple loop: < 1us
- CompactRIO with I/O communication: < 10us
- Factors: CPU load, memory bandwidth, interrupt handling

### CPU Core Isolation

NI Linux RT supports core isolation:
- Real-time tasks pinned to dedicated cores
- Non-RT tasks (network, logging) on other cores
- Eliminates scheduler interference on RT cores

### Watchdog Timer

- Monitors loop execution timeout
- Hardware watchdog on RT controllers
- Configurable timeout period and action (reset, safe state, error log)

## 4.2 LabVIEW FPGA Module

LabVIEW FPGA compiles graphical code directly to FPGA hardware logic:

### Compilation Flow

1. G code -> Xilinx Vivado / Intel Quartus synthesis
2. Generated bitstream downloaded to FPGA chip
3. Single-Cycle Timed Loop (SCTL) executes entire loop body in one FPGA clock cycle (40MHz=25ns, 80MHz=12.5ns)
4. True hardware parallelism: independent code paths run truly in parallel

### Key Capabilities

| Feature | Specification |
|---------|--------------|
| Single-cycle execution | Complete arithmetic+comparison+logic in 25ns/12.5ns |
| True parallelism | Multiple independent loops, zero interaction latency |
| Direct I/O access | FPGA pins directly to external signals, <1 clock cycle latency |
| Custom hardware protocols | Bit-level custom digital protocols |
| DSP integration | Xilinx DSP48 slices for FIR/FFT |

### Applications

- **HIL simulation**: Nanosecond-precision simulation models (motor, battery, power electronics)
- **High-speed signal processing**: Real-time FFT (1024-point < 10us)
- **Protocol emulation**: LIN/CAN/FlexRay bit-level implementation
- **Hardware triggering**: Multi-module nanosecond synchronized triggering

### FPGA Programming Challenges

- Parallel hardware thinking differs from CPU sequential/dataflow thinking
- Limited resources (LUTs, DSP slices, block RAM) require optimization
- Compilation time: minutes to hours (place-and-route is computationally expensive)
- Debugging is harder: limited signal visibility inside FPGA fabric

## 4.3 TestStand Integration

TestStand is NI's test execution management software:

- **Test sequence orchestration**: Visual test sequences (flowchart-like) with branching, loops, data processing
- **Multi-language step execution**: Calls LabVIEW, C#, Python, C/C++ DLL, .NET, ActiveX/COM
- **Result management**: Automatic data collection, report generation (XML/HTML/ATML/DB)
- **User management**: Operator permissions, electronic signature, audit trail
- **Parallel test**: Multi-UUT batch testing
- **Database integration**: SQL Server/Oracle, SPC support

## 4.4 Shared Variables & Network Communication

### Shared Variables
Network-published variables providing publish/subscribe data sharing:

- **Network-Published Shared Variable**: Multi-node data sharing across network
- **Single-Process Shared Variable**: In-process data sharing
- **RT FIFO**: Real-time data transfer between loops (non-blocking)
- **Network Streams**: Lossless streaming between host and RT target

### Protocol Support

- TCP/UDP (native VIs)
- Serial (VISA)
- WebSocket
- OPC UA (via NI OPC UA Toolkit)
- MQTT
- HTTP Client
- gRPC (added in 2022)

## 4.6 Real-Time vs Soft Real-Time Distinction

LabVIEW distinguishes three execution tiers:

| Tier | Platform | Jitter | Determinism | GC |
|------|----------|--------|-------------|-----|
| Desktop | Windows host | ms-level | None | Yes |
| RT | NI Linux RT | <10us | Bounded | No |
| FPGA | FPGA fabric | <1 clock cycle | Absolute | N/A |

This three-tier model validates AUDESYS D19 multi-language strategy:
- RT data path (Rust exclusive, <1us): LabVIEW FPGA equivalent
- I/O communication (Rust/C++ + FlatBuffers over UDS, ~10us): LabVIEW RT equivalent
- Control plane/HMI (15 languages over Zenoh, ~100us): LabVIEW host equivalent
