# Chapter 1: Product & Historical Context

## 1.1 Company Background

NI (National Instruments) was founded in 1976 in Austin, Texas by James Truchard, Jeff Kodosky, and Bill Nowlin. All three worked at the University of Texas Applied Research Lab. NI began with GPIB (IEEE 488) interface boards for instrument control.

**Acquisition**: Emerson acquired NI for $8.2 billion in October 2023. NI became Emerson's Test & Measurement division. Headquarters stayed in Austin. Ritu Favre (former NI executive) leads the business group.

**Revenue**: $1.66 billion (2022), 40+ countries, ~35,000 customers.

## 1.3 Virtual Instrumentation

LabVIEW's core philosophy: **Virtual Instrumentation** — use software to replace physical instruments.

A VI (Virtual Instrument) has three components:

1. **Front Panel**: User interface with Controls (inputs: knobs, buttons) and Indicators (outputs: graphs, LEDs). Simulates a physical instrument panel.
2. **Block Diagram**: Graphical source code with function nodes, wires, structures, and terminals. Core of dataflow programming.
3. **Connector Pane**: Defines VI input/output interface (like a function signature). Used when VI acts as SubVI.

This three-layer separation is LabVIEW's fundamental architectural difference from text-based languages: UI, logic, and interface definition are built into the programming paradigm itself.

## 1.4 Dataflow Programming Model

LabVIEW's execution model is dataflow-driven, not control-flow-driven:

- **Data availability drives execution**: A node executes when all its inputs have data. Implicit parallelism — nodes without data dependencies execute simultaneously.
- **Wires define dependencies**: Connections between nodes are both data channels and execution order definers.
- **Parallelism is implicit**: The compiler automatically identifies parallel nodes and schedules them across CPU cores.

**Text vs Dataflow comparison**:

| Feature | LabVIEW (Dataflow) | Text Languages (Control Flow) |
|---------|-------------------|-------------------------------|
| Execution order | Wire-defined data dependencies | Statement order defines control flow |
| Parallelism | Implicit (compiler finds parallel nodes) | Explicit (async/await, pthread, TPL) |
| Loops | While/For Loop structures | while/for statements |
| Conditionals | Case Structure | if-else/switch |
| State persistence | Shift Register | Local/member variables |
| Code reuse | SubVI (like function call) | Function/method call |

Dataflow suits: data acquisition, test sequence automation, real-time control, parallel measurement.

## 1.7 Current Status (2026)

- **License**: Commercial (subscription + perpetual). Community Edition free for non-commercial use.
- **Community**: Millions of LabVIEW users globally. NI official forums, LAVA community, VIPM package manager.
- **Certification**: CLAD -> CLD -> CLA (Associate -> Developer -> Architect).
- **Tools Network**: 2000+ third-party toolkits and plugins.
- **Key trend**: Emerson post-acquisition integration, Nigel AI assistant, continued quarterly releases.
