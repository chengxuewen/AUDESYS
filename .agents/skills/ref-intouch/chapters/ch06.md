# Ch06: 产品特色 — Product Features

## 6.1 ArchestrA 面向对象范式 — 行业首创

ArchestrA 是 InTouch 最核心的技术差异化优势。2002 年推出时，整个 HMI/SCADA 行业仍停留在平面 Tag 模型层面。

- **模板化工程**：设备/过程建模为可复用对象模板，一个"泵"模板包含所有属性、脚本逻辑和图形表示。
- **继承**：派生模板继承父模板全部属性，可覆盖特定属性。
- **包容**：对象包含子对象，形成自然的过程层次结构。
- **版本管理**：模板变更时，所有实例可手动或自动继承变更。

**对行业的影响**：Rockwell 的 Add-On Instructions (AOI) 和 PlantPAx、Siemens 的 TIA Portal Library 和 Typicals 均可追溯至 ArchestrA 的概念影响。

## 6.2 Galaxy 统一配置库

解决了传统 SCADA 工程的"配置分散"问题：
- **单一数据源**：所有工程设计数据存储在一个 SQL Server 数据库中
- **多工程师协作**：检出/检入机制允许多工程师同时工作
- **逻辑命名空间**：所有对象引用使用逻辑名称，而非物理地址
- **部署模型**：配置变更通过部署工作流推送

## 6.3 最悠久的工业 HMI 传承

InTouch 从 1989 年起累积的工程经验：
- 超过 35 年的现场验证
- 庞大的已部署应用库（10-20 年前创建的应用仍在运行）
- 全球数以万计的工程师熟悉 WindowMaker 和 QuickScript

## 6.5 通信驱动覆盖广度

InTouch 拥有业内最广泛的设备通信驱动库，覆盖几乎所有主流 PLC 品牌（AB、Siemens、Omron、Mitsubishi、Schneider、GE、ABB 等）和工业协议（Modbus、PROFIBUS、EtherNet/IP、PROFINET 等）。

## 6.6 从 Legacy 到 Modern 的迁移故事

InTouch 正在经历历史性的现代化转型：
- WindowViewer（1990 年代 Win32） → OMI（2020 年代 HTML5）
- 仅 Windows Desktop → 多设备（桌面 + Web + 移动）
- 平面 Tag → 对象模板（ArchestrA）
- 固定许可阶梯 → 无限/订阅许可（Unlimited）
- 传统 QuickScript → QuickScript.NET + C# 脚本
- 封闭导出 → OPC UA Server + MQTT
