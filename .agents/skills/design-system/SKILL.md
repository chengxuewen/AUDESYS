# AUDESYS 设计系统技能

## 1. 使命 (Mission)

为 AUDESYS 工业自动化控制平台生成一致、安全、高效的用户界面。

**用途**：引导 AI 代理在前端开发中遵循 AUDESYS 工业设计系统，确保所有 UI 输出在色彩、字体、间距、组件和交互行为上与平台规范一致。

**适用范围**：`apps/debug/` 中所有前端工作，以及未来 `apps/frontend/` 中的开发任务。

---

## 2. 品牌 (Brand)

AUDESYS 的品牌形象由工业控制场景的根本需求决定：

- **工业自动化控制平台** — 面向机器人控制与生产线监控，非消费级产品
- **安全优先** — 错误状态必须醒目，运行状态必须清晰。视觉设计直接服务于操作安全
- **数据密集** — 每屏信息量最大化。间距紧凑，避免大面积留白。像素服务于数据

设计语言根植于工业控制室的实际使用环境：暗光、长时间注视、高信息密度、快速视觉扫描。

---

## 3. 风格基础 (Style Foundations)

### 3.1 暗色优先

默认主题为深色画布（`#010102`），模拟工业控制室的低光环境。亮色主题仅作为辅助选项，不得作为默认主题使用。

### 3.2 Linear 风格基底 + 工业强调色

采用 Linear 风格的克制四阶表面层叠结构，从深到浅递增层级感：

| 层级 | 典型色值 | 用途 |
| --- | --- | --- |
| canvas | #010102 | 最底层画布，页面背景 |
| surface-1 | #0A0A0B | 默认面板背景 |
| surface-2 | #141416 | 卡片、表单区域 |
| surface-3 | #1E1E22 | 导航栏、侧边栏 |
| surface-4 | #2A2A30 | 悬浮态、选中态背景 |

### 3.3 强调色

- **主色（工业琥珀色 #FFB800）**：操作引导、关注焦点、激活状态。在暗色背景上具备高对比度又不刺眼，适合长时间注视
- **辅助色（安全蓝 #0066CC）**：确认操作、稳定状态、信息提示。与 ANSI Z535 工业安全标准一致

> **详细色板**见 `docs/architecture.md` §五、前端架构 §2 色彩系统。

## 4. 色彩使用规则 (Color Usage Rules)

语义状态色承载操作安全含义，使用场景必须严格定义：

### 4.1 状态色定义

| 语义 | 色值 | 对应场景 |
| --- | --- | --- |
| operational (运行/正常) | #00D26A | 设备运行中、数据正常、操作成功 |
| warning (警告) | #FFB800 | 参数异常、接近阈值、需要关注 |
| error (错误/故障) | #FF4444 | 设备故障、通信中断、数据异常 |
| info (信息) | #0099FF | 系统消息、状态变更、提示信息 |

### 4.2 使用规则

- **operational (#00D26A)**：仅用于表示设备或系统处于预期工作状态。不可用作装饰色
- **warning (#FFB800)**：表示需要操作员关注但尚未达到危险程度的状态
- **error (#FF4444)**：仅用于需要立即干预的故障状态。红色不应出现在任何正常状态的界面元素上
- **info (#0099FF)**：用于中性信息提示，不可与操作引导混淆
- **优先级**：当多个状态色同时出现在一个区域时，优先级顺序为：error > warning > info > operational

### 4.3 禁止项

- ❌ 使用装饰性渐变或彩虹色方案
- ❌ 在非状态指示器中使用红/绿色（防止与语义状态混淆）
- ❌ 在正常状态中使用 #FFB800 或 #FF4444（保留给警示/故障）

## 5. 字体规则 (Typography Rules)

### 5.1 字体选择

| 角色 | 字体 | 回退 | 用途 |
| --- | --- | --- | --- |
| 正文 | Geist Sans | -apple-system, system-ui | UI 标签、按钮、导航、描述 |
| 等宽 | JetBrains Mono | Menlo, monospace | 设备数据、日志、代码、数值 |

### 5.2 使用规则

- **Geist Sans**：所有界面标签、导航文字、按钮、操作说明、栏目标题
- **JetBrains Mono**：所有数据值、日志条目、代码片段、技术标识符、主题名称
- **关键约束**：数据密集界面中的数值列必须使用等宽字体确保跨行对齐

### 5.3 字号体系

字号体系紧凑，针对高密度数据展示优化（20px → 16px → 14px → 13px → 12px），下降幅度小，避免面板间切换时的视觉跳跃。

## 6. 组件指南 (Component Guidelines)

以下为工业组件视觉规范。AI 代理在实现组件时应遵循以下定义。

### 6.1 状态面板 (StatusPanel)

- 固定宽度卡片布局，标签在上、数值在下
- 背景使用 `--color-surface-1`，左侧 3px 语义状态色条
- 数值使用 JetBrains Mono 等宽字体，20-24px
- 悬浮态背景提升至 surface-2，警告态边框切换为 #FFB800，故障态边框切换为 #FF4444

### 6.2 数据表格 (DataTable)

- 表头 13px 灰色 #A0A0B0，表体 13px #E8E8ED
- 行高 36px（紧凑模式）/ 44px（标准模式）
- 数值列右对齐 + JetBrains Mono，状态列固定宽度 + 语义色标签
- 行选中态左侧 2px #FFB800 标记

### 6.3 日志查看器 (LogViewer)

- JetBrains Mono 12px，画布背景 (#010102) 模拟终端风格
- 行高 20px 最大化行密度，自动滚动到底部 + 暂停滚动切换
- 警告行前缀 #FFB800 色条，错误行前缀 #FF4444 色条

### 6.4 主题树 (TopicTree)

- 节点高度 28px，缩进 16px 每级
- 主题前缀色按模块着色（如 `/log` → #0099FF，`/system` → #00D26A）
- 叶子节点显示数据类型标签如 `[string]` `[json]`

### 6.5 实时数据流 (RealtimeStream)

- 多列网格布局，每列一个数据点
- 数值变更时短暂高亮（300ms #00D26A 淡出）
- 过期态 #FFB800 警告色标注，断连态显示 `---` + #FF4444

### 6.6 通用约束

- 圆角不超过 8px（`--radius-sm`: 4px, `--radius-md`: 6px, `--radius-lg`: 8px）
- 阴影仅用于浮动元素（下拉菜单、弹出框、模态框），从不用于面板分区
- 面板分区使用 1px 发丝边框（`--color-border-default`），边框优先于阴影
- 间距以 4px 为基本单位，内部间距 4-8px，组件间距 16-24px

## 7. 无障碍 (Accessibility)

### 7.1 对比度目标

| 层级 | 标准 | 应用范围 |
| --- | --- | --- |
| 文字对比度 | WCAG AAA（7:1） | 所有正文、状态标签、数值 |
| 大文字对比度 | WCAG AAA（4.5:1） | 字号 >= 18px 的标题 |
| 非文字对比度 | WCAG AA（3:1） | 图标、边框、状态指示器 |
| 状态色对比度 | WCAG AA（3:1） | 语义状态色在背景色上 |

操作员可能在远距离（1-2 米外）或视线偏移情况下瞥见界面，低对比度内容无法被快速识别。工业控制界面不应妥协对比度。

### 7.2 触控目标

- 按钮（主要操作）：40px x 32px 最小尺寸
- 图标按钮：32px x 32px
- 开关/滑块：32px x 20px

### 7.3 焦点指示

- 键盘焦点指示器必须始终可见：使用 2px #FFB800 外发光环
- 焦点顺序必须遵循视觉布局（Tab 键导航从左到右、从上到下）
- 所有可交互元素必须可以通过键盘访问

### 7.4 色彩非依赖

语义信息不能仅通过颜色传达。状态色必须配合图标、文字标签或图案使用：

- 错误状态：红色 + 错误图标 + "故障"文字标签
- 运行状态：绿色 + 运行图标 + "运行中"文字标签
- 警告状态：琥珀色 + 警告图标 + "警告"文字标签

## 8. 应做与不应做 (Do's & Don'ts)

### ✅ 应当做 (DO)

- 使用暗色主题作为默认主题（canvas #010102）
- 使用 4px 基线网格控制所有间距
- 使用语义状态色准确传达设备状态
- 在数据密集界面使用等宽字体（JetBrains Mono）
- 为每个语义状态色提供非色彩标识（图标 + 文字）
- 保持圆角 <= 8px，阴影最小化
- 使用发丝边框而非投影来划分区域
- 确保状态色在深色背景上满足 WCAG AAA 对比度
- 数值变更动画控制在 300ms 以内且仅使用淡出
- 使用 #FFB800 琥珀色作为主要操作引导色

### ❌ 不应做 (DON'T)

- 使用亮色主题作为默认主题
- 使用大块留白或装饰性插图填充空间
- 使用消费级设计中的大圆角（>= 12px）或大阴影
- 在警告/错误状态中仅使用颜色传达含义（必须配合文字/图标）
- 使用骨架屏、旋转加载器、进度条动画
- 在非状态指示器中使用红/绿色（防止与语义状态混淆）
- 使用低于 WCAG AA 标准的文字对比度
- 在正常状态中使用 #FFB800 或 #FF4444（保留给警示/故障）
- 引入装饰性动效或过渡动画
- 使用自动消失的错误状态（工业场景中错误必须持续可见直到操作员确认）
- 使用消费级间距（间距过松不满足工业数据密度要求）
- 创建设计系统之外的色值（所有颜色必须来自本文档定义的色板）
- 使用 Tailwind CSS v3 配置语法（项目使用 v4）

## 9. 动效原则

### 允许的动画

| 场景 | 时长 | 缓动函数 |
| --- | --- | --- |
| 面板展开/收起 | 200ms | ease-out |
| 状态变更背景色过渡 | 150ms | ease-out |
| 悬浮态切换 | 100ms | ease-out |
| 数值更新高亮淡出 | 300ms | ease-out |
| 模态框出现 | 200ms | ease-out |

### 禁止的动画

- 视差滚动效果
- 骨架屏动画
- 装饰性旋转、弹跳、渐变移动
- 背景视频或动态壁纸
- 图表进入动画（数据应立刻出现）

### 高刷场景规则

每秒更新超过 10 次时禁用高亮动画，超过 30 次时使用 `requestAnimationFrame` 机制。

## 10. 响应式策略

AUDESYS 采用桌面优先的固定面板布局：

| 面板类型 | 宽度 | 说明 |
| --- | --- | --- |
| 主内容区 | 自适应填充 | 占据剩余空间 |
| 侧边栏 | 240px | 导航、树形结构 |
| 属性面板 | 320px | 详情、编辑表单 |
| 状态栏 | 100% x 32px | 底部全局状态 |

### 不适配场景

- 不针对手机横屏/竖屏适配
- 不针对平板竖屏适配（目标屏幕 14 寸及以上）
- 不针对浏览器窗口无限缩小（最小宽度 1280px）
