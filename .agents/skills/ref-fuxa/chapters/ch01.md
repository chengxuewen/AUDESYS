## 1.2 Core Value Proposition

FUXA positions itself as a **completely open-source Web-native SCADA/HMI platform**:

- **Zero cost to start**: MIT license, no licensing fees for any use case including commercial deployment
- **Fully Web-based**: All engineering and runtime monitoring in the browser. No desktop client installation required
- **Lightweight architecture**: Node.js + Angular stack, runs on Docker, Linux, Windows, macOS, Raspberry Pi
- **Drag-and-drop SVG editor**: No Web coding required to create industrial monitoring screens — lowers barrier for automation engineers unfamiliar with frontend development
- **Multi-protocol support**: Native Modbus RTU/TCP, OPC UA, MQTT, Siemens S7, BACnet, EtherNet/IP, WebAPI
- **REST API**: Complete REST API for integration with MES, ERP, AI analytics platforms

FUXA occupies in open-source SCADA a position analogous to Grafana in monitoring visualization — using modern Web technologies to lower the barrier for professional visualization.

## 1.4 Business Model

FUXA is a **completely free MIT open-source project** with no commercial license model:

- **Source**: GitHub public repo, anyone can fork/modify/distribute
- **npm install**: `npm install @frangoteam/fuxa`
- **Docker install**: `docker pull frangoteam/fuxa:latest`
- **Revenue sources (speculated)**: GitHub Sponsors, enterprise customization services, paid technical support, cloud hosting (future)

This contrasts sharply with Ignition Maker Edition (free for non-commercial only) or traditional commercial SCADA — FUXA welcomes any form of commercial use.

## 1.5 AUDESYS Context

FUXA is the most practical reference for AUDESYS Studio's HMI Designer. Unlike CODESYS (closed-source, expensive) or LabVIEW (binary .vi format), FUXA's JSON-native, Web-first approach aligns with AUDESYS D24 (YAML + FlatBuffers) and D21 (Tauri + React + TypeScript) decisions.
