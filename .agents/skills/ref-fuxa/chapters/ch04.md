## 4.3 Deployment Methods

| Method | Command/Approach | Use Case |
|--------|-----------------|----------|
| Docker | `docker run -d -p 1881:1881 frangoteam/fuxa:latest` | Quick deploy, production |
| npm install | `npm install @frangoteam/fuxa` | Existing Node.js environment |
| Source install | `git clone` + `npm install` + `npm start` | Development/customization |
| Electron app | Self-built (README mentions Electron option) | Desktop application use |
| Raspberry Pi | npm install or Docker (ARM image) | Edge/embedded |

- **Default port**: 1881
- **Config file**: `_project` directory (project files, JSON format)
- **Database file**: `fuxa.db` (SQLite config database)

## 4.6 Key Ecosystem Insights for AUDESYS

1. **FUXA is the most active open-source Web SCADA** — but production readiness is only "medium" vs Rapid SCADA/OpenSCADA's "high"
2. **Single-maintainer risk**: unoceelli is the primary developer — bus factor = 1
3. **Community is GitHub-driven** — no dedicated forum or active mailing list
4. **Deployment is Docker-first** — aligns with AUDESYS Phase 1 deployment strategy
