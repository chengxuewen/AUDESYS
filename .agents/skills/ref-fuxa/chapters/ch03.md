# Chapter 3: Feature Overview (Visualization, Alarms, Trends, REST API)

## 3.1 Visualization

FUXA's core capability is **SVG-based Web HMI visualization**.

### SVG Editor Features

- **Drag-and-drop design**: Drag components from library to canvas
- **Component types**:
  - **Basic shapes**: Rect, rounded rect, circle, ellipse, line, polyline, polygon, path
  - **Text**: Single-line, multi-line, styled labels
  - **Gauge/Indicator**: Circular gauge, bar gauge, tank level, linear gauge
  - **Control**: Button, toggle switch, slider, input field
  - **Chart**: Real-time trend, history trend, pie chart, bar chart
  - **Container**: Panel, embedded iframe, pipe
  - **Media**: Image, SVG icon, camera video stream
  - **Data table**: Real-time data table
  - **HTML component**: Custom HTML/CSS
- **Data binding**: Component properties (color, size, text, visibility) bind directly to Tag values
- **Conditional styling**: Value-based color/style changes (e.g., temperature > 100°C → red)
- **Animation**: Rotation, movement, scaling (bindable to Tag values)
- **Event handling**: Mouse click/hover events trigger actions (navigation, write Tag, execute script)
- **Layer management**: Z-Order sorting, grouping, locking
- **Alignment tools**: Snap-to-grid, distribute, align
- **Zoom/Pan**: Editor canvas supports zoom and pan

### Real-time HMI Screen

- **Full-screen mode**: Runtime screens can be displayed fullscreen (suitable for large screens/TVs)
- **Auto-refresh**: WebSocket real-time Tag updates, no manual refresh needed
- **Multi-page navigation**: Navigation between multiple view pages
- **Mobile access**: Responsive layout, supports phone/tablet browsers

## 3.2 Alarm Management

- **Alarm definition**: Limit-based alarms from Tag values (high, high-high, low, low-low)
- **Alarm levels**: Info, Warning, Alert, Critical (4 levels)
- **Alarm states**: Active, Acknowledged, Cleared
- **Alarm acknowledgment**: User can acknowledge alarms
- **Alarm notification**: Pending (possibly via Webhook or email)
- **Alarm history**: Alarm records stored in SQLite, queryable
- **Alarm sound**: Alarm sound alerts supported

## 3.3 Trends & Historical Data

- **Real-time trend**: Real-time trend chart embedded in SVG screen
- **History trend**: Query historical data and display as trend chart
- **Data storage**:
  - Default: SQLite (suitable for small-scale data)
  - Optional: InfluxDB (high-performance time-series, large-scale)
- **Data export**: Pending confirmation (possibly CSV export)
- **Data playback**: Pending confirmation

## 3.4 Reports

FUXA v1.3.x introduces report functionality:
- **Report types**: Daily/weekly/monthly, custom time range
- **Output format**: PDF, CSV (pending confirmation)
- **Scheduled generation**: Automatic report generation by schedule
- **Data source**: From Tag historical data or alarm data

## 3.5 Recipe Management

- **Recipe definition**: A preset set of Tag values
- **Recipe operations**: Load, save, edit
- **Recipe storage**: SQLite database

## 3.6 User & Role Management

- **User management**: Create, edit, delete users
- **Role types**: Admin (full control), Editor (edit projects, not users), Viewer (read-only)
- **Permission application**:
  - Different users access different HMI projects
  - Control operations (write Tag, acknowledge alarm) require appropriate permissions
- **Session management**: JWT token expiration mechanism

## 3.7 Multi-language Support

- **Built-in languages**: English (primary), partial i18n infrastructure exists
- **Custom translation**: Frontend translation files modifiable (pending confirmation)

## 3.8 REST API

FUXA provides a complete REST API, the core of its enterprise integration capability:

| API Endpoint | Function |
|-------------|----------|
| `GET /api/tags` | Get all Tag list |
| `GET /api/tags/{id}/value` | Read Tag current value |
| `POST /api/tags/{id}/value` | Write Tag value |
| `GET /api/alarms` | Get current alarm list |
| `POST /api/alarms/{id}/ack` | Acknowledge alarm |
| `GET /api/history` | Query Tag historical data |
| `GET /api/projects` | Get project list |

The REST API enables FUXA to act as a data middleware, exposing industrial data to upper-level MES/ERP systems.
