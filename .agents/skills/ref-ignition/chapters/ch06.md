# Chapter 6: 产品特色 — Ignition Key Differentiators

## 6.1 Unlimited Licensing — Disruptive Business Model

Traditional SCADA charges by "point count × client count":
- Expanding the system increases license costs linearly
- Project managers over-procure or under-estimate
- Integrators have high license overhead squeezing margins

Ignition breaks this: one server license = unlimited tags + unlimited clients + unlimited projects.
- Integrators buy once, reuse infinitely
- End users expand freely without extra licensing cost

This is an **architecture trust signal** — Inductive Automation uses "unlimited" to demonstrate confidence in their platform's scalability.

## 6.2 Web-Launch Technology — Zero-Install Deployment

Traditional SCADA: install client on every operator station → update one by one → version mismatch problems.

Ignition: all clients launch from Gateway → update Gateway = update all clients → guaranteed version consistency.

## 6.3 Perspective — Mobile-First SCADA

Perspective (2019) marked a turning point for the SCADA industry:
- **First** mobile-responsive design in industrial SCADA
- Same HMI design adapts to phone, tablet, desktop
- Designer runs in browser (like Figma workflow)
- Session concept upgrades "window" to modern "session"

Industry significance: similar to React Native's impact on mobile development — web tech delivering near-native experience.

## 6.4 MQTT Sparkplug First-Class

Ignition was the first commercial SCADA platform to adopt MQTT Sparkplug as a core protocol:
- **Decentralized**: Pub/sub, no single point of failure
- **Auto-discovery**: Sparkplug Birth certificates auto-detect devices and tags
- **Report by Exception**: Reduces bandwidth
- **State awareness**: Birth/Death certificate mechanism for device online status

## 6.5 Gateway-Centric Architecture

All intelligence in Gateway, clients are "thin terminals":
- Data acquisition in Gateway (not clients)
- Clients receive data via binary protocol (not independent polling)
- Unified security policy and audit logs

This architecture makes Ignition naturally suited for cloud deployment and multi-tier architectures.
