# Chapter 2: 技术特性 — Ignition Technical Architecture

## 2.1 Three-Tier Architecture

```
Client Tier        Gateway Tier           Data Tier
 (Browser/App)      (Java Server)         (PLC/DB/Sensors)
```

### Gateway (Core Service)

Java-based server process — the heart of Ignition:

| Subsystem | Role |
|-----------|------|
| **Web Server** | Jetty/Tomcat (unconfirmed), HTTPS, session management |
| **Tag Engine** | Real-time data engine, manages all Tags from all Providers |
| **Script Engine** | Jython (Python 2.7 on Java) for server-side scripts |
| **Alarm Engine** | ISA 18.2 alarm management, Alarm Pipeline |
| **History Engine** | Time-series data to SQL or Ignition Historian |
| **Report Engine** | JasperReports-based report generation |
| **Security** | Users, roles, security levels, project-level access |
| **Redundancy** | Active-Standby with heartbeat |

Gateway process details:
- Java 11+ (8.3)
- Memory: 2-8 GB typical
- G1GC for large heap optimization
- Dual-NIC for enterprise/control network bridging

### Designer (Engineering IDE)

Cross-platform desktop IDE (Java/Swing or Eclipse RCP):
- Project tree (windows, templates, scripts, reports, transaction groups)
- Tag Browser
- Vision Designer: legacy desktop HMI
- Perspective Designer: web/mobile HMI (browser-based since 8.1)
- Transaction Group Designer: data acquisition config
- Scripting Console: interactive Python

### Vision Module (Desktop HMI)

Traditional desktop HMI, Java Swing rendering:
- Launch via Java Web Start or local install
- 100+ standard components
- Popup/docked windows, multi-monitor
- Window templates
- Property Binding for data
- Python event scripts
- **Limitation**: Non-responsive, Java runtime required, legacy

### Perspective Module (Web/Mobile HMI)

Next-gen HMI (2019, Ignition 8.0) — Ignition's future direction:

| Aspect | Detail |
|--------|--------|
| **Frontend** | React/TypeScript (inferred), WebSocket with Gateway |
| **Designer** | Fully browser-based (like Figma workflow) |
| **Layout** | Flexbox + CSS Grid responsive |
| **Session** | Per-tab lifecycle + properties |
| **Components** | 200+ built-in (charts, gauges, maps, alarm tables, QR) |
| **Data Binding** | Bidirectional, transforms, expression binding |
| **Scripting** | Python (server) + JavaScript (client, limited) |
| **Mobile** | Camera, GPS, push notifications |
| **Offline** | Local cache on disconnect |
| **Power Chart** | Million-point time-series charting |

**Vision vs Perspective**:

| Feature | Vision | Perspective |
|---------|--------|-------------|
| Render | Java Swing | HTML5/React |
| Designer | Desktop app | Browser |
| Mobile | No | Native responsive |
| Components | 100+ | 200+ |
| Performance | Medium (single-threaded Swing) | High (WebWorker + Canvas) |
| Deployment | JRE required | Zero-install |
| Offline | No | Yes |
| Future | Maintenance | Primary development |

## 2.2 Data Acquisition & Communication

### Tag Provider Architecture

| Provider | Protocol | Notes |
|----------|----------|-------|
| OPC UA | OPC UA TCP | Built-in client, connect any OPC UA server |
| MQTT Engine | MQTT Sparkplug | Auto-discover devices, auto-create tags |
| Modbus | Modbus TCP/RTU | |
| Siemens S7 | S7 Protocol | Direct Siemens PLC |
| Allen-Bradley | EtherNet/IP, CSP | Direct Rockwell Logix |
| SQL Bridge | JDBC | Bidirectional DB sync |
| Memory | None | Internal tags |
| Expression | None | Calculated from other tags |
| OPC DA | DCOM | Legacy (Windows only) |
| BACnet | BACnet/IP | Building automation (unconfirmed) |

### MQTT Sparkplug First-Class

Ignition pioneered MQTT Sparkplug as a core protocol:
- **MQTT Engine**: Subscribe, auto-discover devices (NBIRTH/DBIRTH), auto-create tags
- **MQTT Distributor**: Publish internal tags as Sparkplug
- **MQTT Transmission**: Advanced publishing with custom payload

Auto-discovery flow: MQTT Broker → Sparkplug → MQTT Engine → Auto-create Tag → Tag Engine

### OPC UA Integration

- Built-in OPC UA Client: Data Access, Alarms & Conditions, Historical Access
- Built-in OPC UA Server: Expose tags as OPC UA address space
- Security: None, Basic128Rsa15, Basic256, Basic256Sha256, Aes128Sha256RsaOaep, Aes256Sha256RsaPss
- Auth: Anonymous, username/password, certificate

## 2.3 Scripting & Expression

### Jython (Python 2.7 on Java)

Script execution contexts:
- **Gateway scripts**: Startup, scheduled background tasks
- **Client scripts**: Event-driven (button click, tag change, window open)
- **Tag event scripts**: On tag value change
- **Alarm pipeline scripts**: Custom alarm processing
- **Transaction group scripts**: Data acquisition/forwarding

Script API access:
- `system` library: core API (tag read/write, DB, alarms)
- Java class libraries via Jython
- Third-party JARs

### Expression Language

Simplified syntax for computed tags:
```
{tag1} + {tag2} * 1.5
if({temperature} > 100, "HIGH", "OK")
```

## 2.4 Security Model

### Authentication
- Local users, Active Directory/LDAP
- Federated Identity: SAML, OAuth 2.0, OpenID Connect (Okta, Azure AD, Google)
- 2FA via identity provider

### Authorization
- Security Levels (hierarchical, replaces roles)
- Project-level, component-level, tag-level access control
- Source IP restrictions
- Guest mode (limited, write-protected)

### Communication Security
- TLS/SSL for Perspective (HTTPS + WSS)
- OPC UA Sign & Encrypt

## 2.5 Scalability Architecture

| Pattern | Description | Typical Use |
|---------|-------------|-------------|
| **Basic** | Single Gateway | Small-medium factory |
| **Scale-Out** | Multi-Gateway cluster (horizontal) | High-concurrency (thousands of clients) |
| **Hub and Spoke** | Central + remote edge Gateways | Multi-site geo-distributed (oil/gas, water) |
| **Enterprise** | Multi-tier (plant → region → enterprise) | Large multinational |
| **Cloud-Based** | Gateway on AWS/Azure/GCP | Cloud/hybrid |

Scale-Out: Frontend load-balancer Gateways + Backend data Gateways.

## 2.6 Web-Launch Technology

Users access Gateway via browser (`http://gateway:8088`):
- Vision: Java Web Start (JNLP) auto-download
- Perspective: Renders directly in browser
- No manual client install, version managed by Gateway
- Binary protocol or WebSocket for real-time communication
