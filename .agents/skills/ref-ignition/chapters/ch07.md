| Ignition Perspective Design | AUDESYS Studio Reference Point |

## 7.7 Script Language Strategy

Ignition chose Python (Jython). Lessons for AUDESYS:
- Python has high engineer familiarity
- **Jython problem**: Python 2.7 EOL, migration to GraalPy/Python 3 is slow
- AUDESYS should consider **WASM plugins** (language-agnostic) or **Lua** (lightweight embed)

## 7.9 Web IDE Architecture

From Ignition Perspective Designer:
- Perspective Designer runs fully in browser — traditional industrial IDEs (TIA Portal, CODESYS, TwinCAT) are all desktop apps
- Ignition proved **industrial SCADA engineering can be done in a browser**
- **Recommended strategy**: HMI design fully web-based, PLC programming desktop-first (LSP + syntax highlighting in web still has limitations in industrial IDEs)

## 7.10 Pitfalls to Avoid

1. **Vision ↔ Perspective dual maintenance**: Two HMI product lines double maintenance. AUDESYS should pick ONE tech stack from day one.
2. **Jython (Python 2.7) lock-in**: Script language runtime becomes 10-year technical debt. Choose embedding strategy carefully.
3. **Java dependency**: Gateway on JVM is heavy for embedded/RT scenarios. AUDESYS Runtime should use C/Rust for real-time paths.
4. **MQTT vendor dependency**: MQTT modules depend on Cirrus Link. Core communication should be in-house or fully open-source.
